# Contributing to AirGap v2

We welcome contributions to AirGap v2! As a project focused on security and performance, we have a few guidelines to ensure the codebase remains robust.

## 🛠️ Development Workflow

1.  **Fork the repository** and create your branch from `main`.
2.  **Install dependencies** using `npm install --legacy-peer-deps`.
3.  **Ensure all tests pass** before submitting a PR: `npm run test`.
4.  **Adhere to the coding style**: We use Prettier and ESLint. Run `npm run lint` to check for issues.
5.  **Use specific 2026 standards**: React 19, Vite 7, and Tailwind 4 features are encouraged.

## 🔒 Security Policy

If you discover a security vulnerability, please do NOT open a public issue. Instead, report it privately.

## 🤝 Code of Conduct

Always remain respectful and professional in all interactions within the community.

## 📄 Pull Request Process

- Update the documentation for any new features.
- Ensure the CI pipeline passes.
- PRs will be reviewed by the maintainers before merging.

*Happy coding!*
